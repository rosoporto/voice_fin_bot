# Voice Finance Bot

MVP Telegram-бота для учета доходов и расходов из текста, голосовых сообщений и txt-файлов.

## Запуск

1. Установите зависимости:

```bash
uv sync
```

2. Создайте `.env` на основе `.env.example`.
3. Положите JSON ключ сервисного аккаунта Google рядом с проектом или укажите путь в `GOOGLE_SERVICE_ACCOUNT_FILE`.
4. Дайте сервисному аккаунту доступ к Google Таблице.
5. Укажите Telegram ID разрешенных пользователей в `ALLOWED_USER_IDS` через запятую. Если список пустой, бот заблокирует всех.
6. Запустите:

```bash
uv run main.py
```

Для PM2:

```bash
pm2 start "uv run main.py" --name fin-bot
```

## Использование

Бот принимает голосовые сообщения, текстовые операции через `/set кофе 250` и txt-файлы со строками вида `название сумма`. При загрузке txt-файла бот сразу сообщает, что начал обработку, а затем возвращает итог по всем найденным операциям. Если расход нужно отнести к бизнесу, начните строку с `Бизнес:`, например `Бизнес: оплата хостинга 800`; такая операция будет записана в категорию `бизнес`.

Для аналитики доступны `/stats` и `/balance`. Если вы вручную меняете категории на листе «Справочник» в Google Таблице, выполните `/refresh_categories`: бот заново прочитает список категорий из таблицы и заменит кэш Redis, чтобы следующие запросы LLM учитывали ваши правки.
